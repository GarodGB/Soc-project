Workflow

From idea to validated detection

A consistent, repeatable process for every rule your team writes — from first draft to production deployment.

01
Write the detection
Create a Sigma rule, map it to a MITRE technique, set a category (Windows, Linux, Identity), and document known false positives.
02
Map required telemetry
Define which log sources the rule depends on. Confirm those sources are healthy and actively collecting before deploying.
03
Validate against samples
Run the rule against TP and TN sample events. Measure precision and recall. Only promote rules that pass validation.
04
Track ATT&CK coverage
Watch your coverage map update in real time as detections go live. Identify gaps and close them systematically.
